OPERATIONAL MECHANISM

The four-stage protocol: Sensing, Processing, Verdict, and System Logging.

1. THE AQUASENTRY 4-STAGE DATAFLOW ARCHITECTURE

STAGE 1: ACQUIRE DATA

The device initializes and performs sequential readings from submerged probes.

  • Input Matrix: pH, TDS, Turbidity, Temp.
  • Action: Takes **N=5** samples over **3s** for redundancy.

STAGE 2: FILTER & PROCESS

Raw sensor readings are refined to eliminate noise and spurious data spikes.

  • Method: Calculates the **Median(N=5)** for a stable reading.
  • Output: Clean data vector for decision logic.

STAGE 3: VERDICT LOGIC

Filtered data is passed through a simple, conservative threshold matrix for safety determination.

  • Logic: Compares against **WHO/Local** safe limits.
  • Output: Status code (0: SAFE, 1: CAUTION, 2: UNSAFE).

STAGE 4: LOG & SLEEP

The final verdict and raw data are recorded, and the system powers down to conserve energy.

  • Output: OLED Display + MicroSD Log.
  • Power State: Enters **deep sleep** for efficiency.

3. VERDICT MATRIX // CONSERVATIVE THRESHOLDS

The decision logic is based on a **conservative matrix** to ensure the highest standard of public safety, using pre-set thresholds from international standards.

PARAMETER SAFE RANGE FUNCTIONALITY
pH Level **6.5 to 8.5** Acidity/Alkalinity Balance.
TDS (Dissolved Solids) < **300 ppm** Overall mineral/contaminant load.
Turbidity < **5 NTU** Suspended solids/cloudiness factor.

OUTPUT STATUS CODES

  • [ 0 ] SAFE: All checks clear. Standard use authorized.
  • [ 1 ] CAUTION: Minor threshold breach. Non-critical use only.
  • [ 2 ] UNSAFE: Critical parameter failure. **IMMEDIATE** shutdown.
